PSG ERP Spec · Cross-module baseline

Compliance baseline

The cross-module floor every PSG-listed vendor answers regardless of which modules they cover. Per the "viewers connect to 1 big checklist — no repeats" rule, the baseline lives here in one place — not duplicated across every module checklist viewer.

What's this for? Mandated by PSG pre-approval. Vendors attest to compliance with Personal Data Protection Act (PDPA), Cyber Essentials Mark (CEM), and Vulnerability Assessment / Penetration Testing (VA/PT). Identical for every module — render once.

☐

VA/PT

6 items ·
☐
VA/PT report dated within 12 months before checklist submission by qualified third party.
☐
VA/PT scope covers network, application, data protection, access control, and API security.
☐
VA/PT assessor holds CREST certification or equivalent recognised security professional credential.
☐
VA/PT report includes executive summary, detailed findings with risk ratings, and remediation recommendations.
☐
Cloud security configuration review included in VA/PT scope if solution uses cloud infrastructure.
☐
Resellers provide VA/PT report from product principal or SOC 2 Type II with technical details.
